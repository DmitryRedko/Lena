cfl::MultiFunction prb::dropLockSwap ( const cfl::Data::Swap &  rSwap,
double  dLowerRate,
double  dUpperRate,

Computes the price of drop-lock swap. The contract behaves as a standard swap with the difference that the first payment time the market swap rate is above dUpperRate or below dLowerRate the fixed rate in the contract is reset to dUpperRate or dLowerRate, respectively, and then remains constant. The fixed rate before the reset time equals rSwap.rate. The market swap rate used for resets corresponds to the swap contract with the same parameters as in the original rSwap, that is, the same number of payments and the period. Note that the initial time is not a reset time.

Parameters
rSwapThe parameters of the underlying interest rate swap.
dLowerRateThe lower value for swap rate after reset.
dUpperRateThe upper bound for swap rate after reset.
rModelThe reference to the implementation of cfl::InterestRateModel.
Returns
The price of the option as the function of the initial values of the state processes in the model.

cfl::MultiFunction prb::futuresOnLibor ( double  dLiborPeriod,
unsigned  iFuturesTimes,

Computes futures price on LIBOR. The set of times, when the futures price is evaluated is given by

\[ t_i = t_0 + i\frac{T-t_0}{M}, \quad i=1,\dots,M, \]

where \(t_0=\) rModel.initialTime(), \(T=\) dFuturesMaturity, and \(M=\) iFuturesTimes. At maturity dFuturesMaturity, the settlement price is

\[ 1 - L(T,T+\Delta), \]

where \(\Delta = \) dLiborPeriod and \(L(s,t)\) is the LIBOR computed at \(s\) for maturity \(t\).

Parameters
dLiborPeriodThe LIBOR period in the futures.
iFuturesTimesThe number of futures times.
dMaturityThe maturity.
rModelThe reference to the implementation of cfl::InterestRateModel.
Returns
The price of the option as the function of the initial values of the state processes in the model.
